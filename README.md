# 我的世界卡牌游戏（Cocos2d-x）

本项目提供一个使用 Cocos2d-x 实现的《我的世界》主题卡牌游戏原型，包括**完整的规则设计**、**卡牌数据**以及**基础场景代码**。道具、怪物、方块与附魔命名均参考《我的世界》既有元素，图片素材建议使用官方风格或相近像素风格进行替换。

## 特色玩法

- **生存回合制**：管理体力、能量与生命值，合理规划采集、战斗、建造与合成。
- **生物群系与天气**：不同生物群系提供专属资源与事件；天气改变战斗与采集效率。
- **合成与装备**：通过配方系统将资源转换为强力装备与建筑，提高战斗力与生存能力。
- **红石与建筑**：建造特殊建筑获得持续增益或触发事件牌效果。
- **Boss 机制**：末影龙与凋灵提供终局挑战，带来更高的风险与回报。

## 运行说明（示例）

1. 创建或使用已有的 Cocos2d-x 3.x 工程。
2. 将 `Classes/` 与 `Resources/` 合并到工程目录。
3. 在 `AppDelegate` 中将启动场景设置为 `GameScene::createScene()`。

## 规则概要

更完整的规则请查看 `RULES.md` 与 `GameRules` 中的文本常量以及 `cards.json` 的卡牌描述字段。

- **卡牌类型**：方块、资源、工具、武器、护甲、怪物、事件、建筑、附魔。
- **基础数值**：生命值、能量、护甲、饥饿值、经验值。
- **胜利条件**：击败末影龙或建造“末地传送门”并激活。
- **失败条件**：生命值归零或连续三回合饥饿为 0。

## 场景策略（Scene Strategy）

为了便于后续扩展，将场景分层为**展示层**、**规则层**与**数据层**：

1. **展示层（GameScene）**
   - 负责 UI 布局、文字展示与用户输入。
   - 只调用规则接口与数据读取结果，不直接写复杂规则。
2. **规则层（GameRules）**
   - 统一维护规则文本与核心规则逻辑的入口。
   - 便于后续将规则拆成 `TurnResolver`、`CombatResolver` 等模块。
3. **数据层（Deck / Cards）**
   - 通过 `cards.json` 描述卡牌与素材路径。
   - 设计师可在不改动代码的情况下增删卡牌或调参。

这种策略能让玩法演进时只改 `GameRules` 和数据，而 UI 与场景最小化改动。
